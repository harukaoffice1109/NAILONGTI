# 微信分享与手机结果图修改清单

## 背景问题

微信内置浏览器中调用 `navigator.share()` 会出现不稳定行为：可能打开“转发给朋友”的联系人界面，但选中联系人后不进入发送确认，也没有真正完成分享。这个能力不应作为微信内的主路径。

同时，网页无法可靠地把生成图片直接写入手机相册。不同环境会把图片下载到文件 App、浏览器下载目录，或者完全不显示保存结果。因此不能承诺“保存到相册”。

## 修改目标

1. 普通浏览器保留原生分享能力。
2. 微信内置浏览器不再调用 `navigator.share()`。
3. 微信内点击“分享给朋友”时，自动复制链接并弹出右上角分享引导。
4. 结果图不再默认下载，统一以预览图为核心，让用户长按保存或截图分享。
5. 结果图预览中保留二维码，二维码内容为 `https://nailongti.pages.dev`。
6. 所有弹窗和按钮适配手机浏览器尺寸。

## 具体修改项

### 1. 分享工具模块

文件：`src/lib/share.ts`

修改：

- 增加 `isWeChatBrowser()`。
- `shareSite()` 返回更明确的状态：
  - `shared`
  - `copied`
  - `wechat-guide`
  - `failed`
- 如果检测到微信环境：
  - 不调用 `navigator.share()`。
  - 直接复制链接。
  - 返回 `wechat-guide`。
- 普通浏览器：
  - 优先调用 `navigator.share()`。
  - 失败或不支持时复制链接。
- `shareImage()` 在微信里直接返回 `unsupported`，不尝试文件分享。

### 2. 结果页分享按钮

文件：`src/main.tsx`

修改：

- 点击“分享给朋友”：
  - 普通浏览器成功：提示“分享面板已打开”。
  - 普通浏览器 fallback：提示“链接已复制，发给朋友开测”。
  - 微信环境：弹出微信引导弹窗。
- 增加状态：
  - `showWeChatGuide`
  - `sharePreview`
  - `notice`
- 保持按钮：
  - `分享给朋友`
  - `生成结果图`
  - `复制链接`
  - `再测一次`
  - `看图鉴`

### 3. 微信分享引导弹窗

文件：`src/main.tsx`

新增组件：`WeChatShareGuide`

内容：

- 标题：`链接已复制`
- 文案：`点击右上角 ···，选择“发送给朋友”或“分享到朋友圈”`
- 辅助提示：`也可以直接截图结果页发送给朋友`
- 按钮：
  - `我知道了`
  - `复制链接`

### 4. 结果图预览弹窗

文件：`src/main.tsx`

修改组件：`SharePreview`

内容：

- 显示生成的结果图。
- 文案改为：
  - `长按图片保存，或直接截图分享`
  - `二维码可打开 nailongti.pages.dev`
- 增加按钮：
  - `复制链接`
  - `关闭`
- 不再使用“下载”作为主要动作。

### 5. 结果图生成逻辑

文件：`src/lib/shareCard.ts`

保持：

- `createShareCardBlob()`
- `createShareCardDataUrl()`
- 二维码在右下角。

检查：

- 结果图二维码仍指向 `https://nailongti.pages.dev`。
- 图片尺寸保持 900x1400。
- 图片可在弹窗中展示。

### 6. 移动端样式

文件：`src/styles.css`

修改/补充：

- 微信引导弹窗样式。
- 结果图预览弹窗按钮样式。
- 手机端弹窗宽度 100%，图片最大高度不超过屏幕。
- 弹窗按钮全宽、可点击区域不小于 48px。
- 避免横向滚动。

### 7. 验证

必须通过：

```bash
npm run check
npm run verify:data
npm run verify:deploy
npm run verify:http
```

浏览器验证：

- 首页能打开。
- 完整答题到结果页。
- 结果页按钮文案正确。
- 点击“生成结果图”出现图片预览。
- 图片预览中有“复制链接”和“关闭”。
- 点击“分享给朋友”在普通浏览器环境能走分享或复制 fallback。
- 模拟微信 UA 时，不调用原生分享，出现微信引导弹窗。
- 页面没有横向溢出。
