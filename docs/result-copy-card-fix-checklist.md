# 结果文案与分享卡排版修改清单

## 用户意图

本次修改不是小幅润色，而是一次结果页内容定位修正：奶龙TI要呈现为“人格测试”，不是“群聊角色测试”。文案需要更像 MBTI 式人格画像，同时保留奶龙抽象感。

同时，生成的结果图存在遮挡问题：标题、描述、tags、二维码区域的纵向空间分配不合理，部分结果如“奶姐”的底部标签会挤压或遮挡描述文字，需要调整 canvas 布局。

## 修改范围

### 1. 结果文案字段

文件：`src/data/types.ts`

只改以下字段：

- `一句话判词`
- `description`
- `tags`

不改以下字段：

- `序号`
- `原SBTI_code`
- `原SBTI人格`
- `奶龙TI_code`
- `奶龙TI类型名`
- `对应情况`
- `图案方向`
- `pattern`
- `image`
- `imageVariants`

原因：算法、图片、隐藏结果分流依赖这些字段，不能在文案修改中引入不必要风险。

### 2. 分享结果卡排版

文件：`src/lib/shareCard.ts`

修改目标：

- 标题不被长类型名撑爆。
- 判词不压住图片。
- 描述不被 tags 或二维码遮挡。
- tags 不再横跨过宽区域，避免碰到二维码。
- 二维码固定右下角，有独立安全区域。
- 描述超出最大行数时用省略号收尾。

## 文案修改标准

### 总体比例

- 70% 人格分析
- 30% 奶龙抽象

### 每条 description 的结构

每条描述按以下结构组织：

1. 人格核心：这个人稳定的心理倾向是什么。
2. 行为模式：遇事如何判断、行动或逃避。
3. 关系模式：与他人相处时的优势和盲点。
4. 奶龙抽象收尾：用一句抽象但贴合人格的奶龙式比喻收束。

### 禁用或压低的表达

除个别确实与社交有关的类型外，尽量避免：

- 群聊
- 群友
- 发给朋友后
- 普通网友
- 把群聊变成直播间
- 全场/群里/群主病作为主叙事

这些词可以极少量出现，但不能成为文案骨架。

### 优先使用的表达方向

- 情绪反应
- 决策方式
- 控制欲/边界感
- 自我保护机制
- 关系中的承担方式
- 压力下的行为
- 行动力与拖延方式
- 安全感来源
- 表达欲与退缩
- 自我评价方式

### 用户点名重点

必须重点修改：

- `奶妈龙`：不能再写成“群聊保育员”，要写成高共情、高责任、过度照顾、边界感不足。
- `奶控龙`：不能再写成“群主/管理群聊”，要写成秩序需求、控制感、流程化、压力下容易接管。

## 逐项执行清单

### A. 审查现有文案

1. 扫描 29 个类型的 `description`。
2. 统计出现“群聊/群友/截图/普通网友”等词的字段。
3. 确认所有模板化 description 都会被替换。

### B. 重写文案

1. 为 29 个类型分别写独立人格画像。
2. 每条 description 控制在约 80-130 个中文字符。
3. 保留类型原有抽象方向，但把主语从“群聊里的你”改为“人格结构中的你”。
4. tags 改为人格标签 + 奶龙抽象标签混合。
5. 一句话判词保留传播感，但尽量更人格化。

### C. 分享卡排版修复

1. 标题字号支持按类型名长度动态缩小。
2. 判词最多 2 行，避免挤压图片。
3. 图片从 440x440 缩小到约 390-400 方形，为描述区释放空间。
4. 描述区域固定在图片下方，有明确最大行数。
5. tags 区域固定在左下角，不和二维码重叠。
6. 二维码固定右下角。
7. `wrapText` 支持省略号，防止最后一行继续往下画。

### D. 验证

必须通过命令：

```bash
npm run check
npm run verify:data
npm run verify:deploy
npm run verify:http
```

浏览器验证：

1. 本地打开首页。
2. 完整答题进入结果页。
3. 生成结果图。
4. 检查结果图弹窗能出现。
5. 检查长文案类型不会遮挡。
6. 重点检查：
   - 奶姐
   - 嘉豪奶
   - 奶妈龙
   - 奶控龙
7. 检查页面无横向溢出。
